# 详细指标解释

## 一、技术分析预测（综合模型）

### 1.1 计算原理

这是一个**加权评分系统**，综合多个技术指标得出最终概率：

**评分规则**（来自 `probability.js`）：
- **基础分**：看涨和看跌各从1分开始
- **价格 vs VWAP**：
  - 价格 > VWAP → 看涨 +2分
  - 价格 < VWAP → 看跌 +2分
- **VWAP斜率**：
  - 斜率 > 0（上升）→ 看涨 +2分
  - 斜率 < 0（下降）→ 看跌 +2分
- **RSI + RSI斜率**：
  - RSI > 55 且 RSI斜率 > 0 → 看涨 +2分
  - RSI < 45 且 RSI斜率 < 0 → 看跌 +2分
- **MACD**：
  - MACD柱状图 > 0 且扩大 → 看涨 +2分
  - MACD柱状图 < 0 且扩大 → 看跌 +2分
  - MACD线 > 0 → 看涨 +1分
  - MACD线 < 0 → 看跌 +1分
- **Heiken Ashi**：
  - 连续2根以上绿色K线 → 看涨 +1分
  - 连续2根以上红色K线 → 看跌 +1分
- **VWAP收复失败**：
  - 如果价格尝试回到VWAP但失败 → 看跌 +3分（强烈信号）

**最终概率计算**：
```
看涨概率 = 看涨得分 / (看涨得分 + 看跌得分)
看跌概率 = 看跌得分 / (看涨得分 + 看跌得分)
```

**时间衰减调整**：
- 随着剩余时间减少，预测概率会向50%收敛
- 公式：`调整后概率 = 0.5 + (原始概率 - 0.5) × (剩余时间 / 总时间)`
- **原因**：时间越少，不确定性越大，预测应该更保守

### 1.2 如何解读

**示例：看涨 46% / 看跌 54%**

- **含义**：模型综合所有技术指标后，认为上涨概率46%，下跌概率54%
- **信号强度**：
  - 如果差距 < 5% → 信号较弱，市场可能震荡
  - 如果差距 5-10% → 信号中等
  - 如果差距 > 10% → 信号较强
- **注意事项**：
  - 这是**技术分析**的预测，不是市场实际定价
  - 需要与Polymarket的实际价格对比，寻找"边缘"（edge）

### 1.3 实际应用

**寻找交易机会**：
1. 如果模型预测看涨60%，但市场只定价40% → 存在做多边缘
2. 如果模型预测看跌70%，但市场只定价50% → 存在做空边缘
3. 如果模型和市场一致 → 可能没有明显边缘

**时间因素**：
- **早期（剩余时间 > 10分钟）**：模型预测更可靠
- **中期（5-10分钟）**：开始应用时间衰减
- **后期（< 5分钟）**：预测快速向50%收敛，不确定性增加

## 二、平均K线（Heiken Ashi）

### 2.1 计算原理

**Heiken Ashi（平均K线）**是一种特殊的K线图，用于过滤市场噪音，更清晰地显示趋势。

**计算公式**（来自 `heikenAshi.js`）：
```
HA收盘价 = (开盘 + 最高 + 最低 + 收盘) / 4
HA开盘价 = (前一根HA开盘 + 前一根HA收盘) / 2
HA最高价 = MAX(原始最高, HA开盘, HA收盘)
HA最低价 = MIN(原始最低, HA开盘, HA收盘)
```

**颜色判断**：
- HA收盘价 ≥ HA开盘价 → 绿色（看涨）
- HA收盘价 < HA开盘价 → 红色（看跌）

**连续计数**：
- 从最新K线开始，向后统计连续相同颜色的K线数量
- 例如：`red x10` 表示连续10根红色K线

### 2.2 如何解读

**颜色含义**：
- **绿色**：上涨趋势，买方占优
- **红色**：下跌趋势，卖方占优

**连续数量**：
- **x1-x2**：趋势刚开始，可能不稳定
- **x3-x5**：趋势确立，信号较强
- **x6-x10**：强烈趋势，动量很强
- **x10+**：极强趋势，但可能接近反转点（物极必反）

**实际应用**：
- **趋势确认**：连续同色K线越多，趋势越可靠
- **反转信号**：连续10根以上同色K线后，出现相反颜色 → 可能反转
- **过滤噪音**：相比普通K线，HA更能过滤短期波动

### 2.3 注意事项

**滞后性**：
- HA使用平均值，反应比普通K线慢
- 可能错过快速反转

**假信号**：
- 在震荡市场中，HA可能频繁变色
- 需要结合其他指标确认

## 三、RSI（相对强弱指标）

### 3.1 计算原理

**RSI（Relative Strength Index）**衡量价格变动的强度和速度。

**计算公式**（来自 `rsi.js`，周期=14）：
```
1. 计算过去14根K线的价格变化
2. 分别统计上涨幅度总和（gains）和下跌幅度总和（losses）
3. 平均涨幅 = gains / 14
4. 平均跌幅 = losses / 14
5. RS = 平均涨幅 / 平均跌幅
6. RSI = 100 - (100 / (1 + RS))
```

**RSI斜率**：
- 计算最近3个RSI值的斜率
- 斜率 = (最新RSI - 3个周期前的RSI) / 2
- 用于判断RSI的变化趋势

### 3.2 如何解读

**RSI数值范围**：0-100

**关键区域**：
- **RSI > 70**：超买区域，价格可能回调
- **RSI 50-70**：强势区域，但未超买
- **RSI 30-50**：弱势区域，但未超卖
- **RSI < 30**：超卖区域，价格可能反弹

**RSI斜率（箭头）**：
- **↑**：RSI上升，动量增强
- **↓**：RSI下降，动量减弱
- **-**：RSI持平

**组合解读**：

| RSI值 | 斜率 | 含义 |
|-------|------|------|
| > 70 | ↑ | 极度超买，强烈看跌信号 |
| > 70 | ↓ | 超买但开始回落，可能回调 |
| 50-70 | ↑ | 强势上升，看涨 |
| 50-70 | ↓ | 强势减弱，谨慎 |
| 30-50 | ↑ | 弱势反弹，可能转强 |
| 30-50 | ↓ | 持续弱势，看跌 |
| < 30 | ↑ | 超卖反弹，可能反转 |
| < 30 | ↓ | 极度超卖，强烈看跌但可能反转 |

**你的日志示例**：RSI 23.1 ↓
- RSI < 30：超卖区域
- 斜率 ↓：RSI仍在下降
- **解读**：极度超卖，但动量仍在下降，这是**矛盾信号**
  - 一方面超卖可能反弹
  - 另一方面下降趋势仍在继续
  - **建议**：等待RSI斜率转正，确认反弹

### 3.3 实际应用

**交易信号**：
1. **超买反转**：RSI > 70 + 斜率转负 → 做空信号
2. **超卖反弹**：RSI < 30 + 斜率转正 → 做多信号
3. **趋势确认**：RSI 50-70 + 斜率向上 → 上升趋势确认

**注意事项**：
- **在强趋势中**：RSI可能长期停留在超买/超卖区域
- **在震荡市场中**：RSI更有效
- **15分钟窗口**：RSI可能反应过快，需要结合其他指标

## 四、MACD（指数平滑移动平均线）

### 4.1 计算原理

**MACD（Moving Average Convergence Divergence）**通过两条EMA的差值来判断趋势。

**计算步骤**（来自 `macd.js`，默认参数：快线12，慢线26，信号线9）：
```
1. 快线EMA = 12周期指数移动平均
2. 慢线EMA = 26周期指数移动平均
3. MACD线 = 快线EMA - 慢线EMA
4. 信号线 = MACD线的9周期EMA
5. 柱状图（Histogram）= MACD线 - 信号线
6. 柱状图变化（histDelta）= 当前柱状图 - 前一个柱状图
```

**关键组件**：
- **MACD线**：快慢EMA的差值，反映短期和长期趋势的关系
- **信号线**：MACD线的平滑，用于产生交易信号
- **柱状图**：MACD线和信号线的差值，反映动量强度
- **柱状图变化**：判断动量是增强还是减弱

### 4.2 如何解读

**柱状图（Histogram）**：
- **> 0（绿色）**：看涨，快线在慢线上方
- **< 0（红色）**：看跌，快线在慢线下方
- **= 0**：快慢线交叉，可能反转

**柱状图变化（扩大/缩小）**：
- **扩大**：柱状图绝对值在增加
  - 正值扩大 → 上涨动量增强
  - 负值扩大 → 下跌动量增强
- **缩小**：柱状图绝对值在减少
  - 正值缩小 → 上涨动量减弱
  - 负值缩小 → 下跌动量减弱

**你的日志示例**：看跌（扩大）
- 柱状图 < 0：看跌
- 扩大：负值在扩大
- **解读**：下跌动量正在增强，这是**强烈的看跌信号**

**MACD交叉**：
- **金叉**：MACD线从下向上穿越信号线 → 看涨信号
- **死叉**：MACD线从上向下穿越信号线 → 看跌信号

### 4.3 实际应用

**交易信号优先级**：
1. **柱状图扩大**：动量增强，信号最强
2. **MACD交叉**：趋势可能反转
3. **柱状图缩小**：动量减弱，谨慎

**组合使用**：
- MACD看跌 + 柱状图扩大 → 强烈看跌
- MACD看涨 + 柱状图扩大 → 强烈看涨
- MACD看跌 + 柱状图缩小 → 下跌可能结束

**注意事项**：
- MACD是**趋势跟踪指标**，在震荡市场中可能产生假信号
- 在15分钟窗口中，MACD可能反应较慢
- 需要结合价格行为确认

## 五、1/3分钟价格变化

### 5.1 计算原理

**价格变化**：计算指定时间窗口内的价格变动。

**计算方法**：
```
1分钟变化 = 当前收盘价 - 1分钟前的收盘价
3分钟变化 = 当前收盘价 - 3分钟前的收盘价

变化百分比 = (价格变化 / 基准价格) × 100%
```

**显示格式**：
```
1分钟变化 | 3分钟变化
例如：-$27.18, -0.04% | -$49.25, -0.07%
```

### 5.2 如何解读

**数值含义**：
- **正值**：价格上涨
- **负值**：价格下跌
- **绝对值**：变化幅度

**时间窗口对比**：
- **1分钟变化**：反映极短期动量
- **3分钟变化**：反映短期趋势

**组合解读**：

| 1分钟 | 3分钟 | 含义 |
|-------|-------|------|
| + | + | 短期上涨趋势 |
| - | - | 短期下跌趋势 |
| + | - | 可能反弹，但整体仍下跌 |
| - | + | 可能回调，但整体仍上涨 |
| 小 | 大 | 趋势加速 |
| 大 | 小 | 趋势减速 |

**你的日志示例**：-$27.18, -0.04% | -$49.25, -0.07%
- 1分钟：下跌$27.18（-0.04%）
- 3分钟：下跌$49.25（-0.07%）
- **解读**：
  - 两个窗口都是负值 → 短期下跌趋势
  - 3分钟跌幅更大 → 下跌在加速
  - 这是**看跌信号**

### 5.3 实际应用

**动量判断**：
- 如果1分钟和3分钟同向 → 趋势确认
- 如果方向相反 → 可能反转

**速度判断**：
- 3分钟变化 > 1分钟变化 → 趋势加速
- 3分钟变化 < 1分钟变化 → 趋势减速

**注意事项**：
- 在15分钟窗口中，1-3分钟的变化可能很敏感
- 需要结合其他指标确认，避免被噪音误导

## 六、成交量加权平均价（VWAP）

### 6.1 计算原理

**VWAP（Volume Weighted Average Price）**是成交量加权的平均价格，反映"真实"的平均成交价格。

**计算公式**（来自 `vwap.js`）：
```
典型价格（TP）= (最高价 + 最低价 + 收盘价) / 3
VWAP = Σ(典型价格 × 成交量) / Σ(成交量)
```

**VWAP斜率**：
- 计算最近N分钟（默认5分钟）的VWAP变化
- 斜率 = (当前VWAP - N分钟前VWAP) / N
- 用于判断VWAP的上升/下降趋势

**价格偏离度**：
- 偏离度 = (当前价格 - VWAP) / VWAP × 100%
- 正值：价格高于VWAP
- 负值：价格低于VWAP

### 6.2 如何解读

**价格 vs VWAP**：
- **价格 > VWAP**：当前价格高于平均成交价，看涨
- **价格 < VWAP**：当前价格低于平均成交价，看跌
- **价格 = VWAP**：价格在平均水平

**偏离度**：
- **偏离度 > 1%**：价格显著高于VWAP，可能回调
- **偏离度 < -1%**：价格显著低于VWAP，可能反弹
- **偏离度接近0**：价格在VWAP附近，相对平衡

**VWAP斜率**：
- **上升**：平均成交价在上升，市场情绪转强
- **下降**：平均成交价在下降，市场情绪转弱
- **持平**：平均成交价稳定

**你的日志示例**：69,584 (-0.16%) | 斜率
- VWAP：$69,584
- 偏离度：-0.16%（价格低于VWAP）
- **解读**：
  - 价格低于VWAP → 看跌信号
  - 偏离度-0.16%不算太大，但仍是负值
  - 需要看斜率判断趋势

**VWAP交叉**：
- 价格从下向上穿越VWAP → 看涨信号
- 价格从上向下穿越VWAP → 看跌信号
- 频繁交叉 → 震荡市场

### 6.3 实际应用

**交易参考**：
- **价格低于VWAP + VWAP斜率下降** → 强烈看跌
- **价格高于VWAP + VWAP斜率上升** → 强烈看涨
- **价格接近VWAP** → 可能反转

**支撑/阻力**：
- VWAP可以作为动态支撑/阻力位
- 价格在VWAP上方 → VWAP是支撑
- 价格在VWAP下方 → VWAP是阻力

**VWAP收复失败**：
- 如果价格尝试回到VWAP但失败 → 这是强烈的看跌信号（在评分系统中+3分）

**注意事项**：
- VWAP是**日内指标**，每天重置
- 在15分钟窗口中，VWAP可能不够敏感
- 需要结合成交量确认

## 七、Polymarket市场数据

### 7.1 市场报价（上涨/下跌价格）

**计算原理**：
- 这些是Polymarket市场的**实际交易价格**
- 通过CLOB（Central Limit Order Book）订单簿获取
- 价格范围：0¢ - 100¢（对应0% - 100%概率）

**价格含义**：
- **上涨价格**：买入"上涨"结果的成本
  - 0¢ = 市场认为上涨概率接近0%
  - 50¢ = 市场认为上涨概率50%
  - 100¢ = 市场认为上涨概率100%
- **下跌价格**：买入"下跌"结果的成本
  - 同理，但方向相反

**价格关系**：
- 理论上：上涨价格 + 下跌价格 ≈ 100¢
- 实际中：由于买卖价差，可能略有偏差

**你的日志示例**：↑ 上涨 0¢ | ↓ 下跌 0.99¢
- 上涨价格：0¢ → 市场认为上涨概率接近0%
- 下跌价格：0.99¢ → 市场认为下跌概率接近99%
- **解读**：市场**极度看跌**，几乎所有人都认为会下跌

### 7.2 流动性（Liquidity）

**含义**：
- 市场中的可用资金量
- 流动性越高，大额交易对价格影响越小

**如何解读**：
- **高流动性（> 10,000）**：容易交易，滑点小
- **中等流动性（1,000-10,000）**：可以交易，但需注意滑点
- **低流动性（< 1,000）**：难以交易，滑点可能很大

**你的日志示例**：10,785
- 流动性较高，可以正常交易

### 7.3 目标价格（Price to Beat）

**含义**：
- 这是市场结算的**参考价格**
- 通常从市场标题或描述中提取
- 例如："Will BTC be above $69,598 at 7:30 AM ET?"

**如何提取**（来自 `index.js`）：
1. 从市场标题中搜索"price to beat"等关键词
2. 提取数字作为目标价格
3. 如果市场开始后价格未设置，使用Chainlink价格作为初始值

**你的日志示例**：$69,598
- 这是结算参考价
- 当前价格 $69,479.06 < 目标价格 $69,598
- **差距**：-$118.93（需要上涨$118.93才能达到目标）

### 7.4 当前价格（Chainlink价格）

**数据来源**：
1. **优先**：Polymarket WebSocket实时价格
2. **次选**：Chainlink WebSocket价格
3. **最后**：Chainlink RPC查询

**价格含义**：
- 这是用于结算的**链上价格**（Chainlink预言机）
- 与币安现货价格可能略有差异

**你的日志示例**：$69,479.06 ↑ (-$118.93)
- 当前价格：$69,479.06
- 箭头 ↑：相对于上次更新上涨（但这是微小变化）
- (-$118.93)：与目标价格的差距

## 八、边缘（Edge）计算

### 8.1 计算原理

**边缘（Edge）** = 模型预测概率 - 市场定价概率

**计算公式**（来自 `edge.js`）：
```
市场上涨概率 = 上涨价格 / (上涨价格 + 下跌价格)
市场下跌概率 = 下跌价格 / (上涨价格 + 下跌价格)

边缘（上涨）= 模型预测上涨概率 - 市场上涨概率
边缘（下跌）= 模型预测下跌概率 - 市场下跌概率
```

**示例**：
- 模型预测：上涨60%，下跌40%
- 市场定价：上涨30¢（30%），下跌70¢（70%）
- 边缘（上涨）= 60% - 30% = +30%（正边缘，可以做多）
- 边缘（下跌）= 40% - 70% = -30%（负边缘，不应该做空）

### 8.2 如何解读

**边缘值**：
- **> 0**：模型认为概率高于市场定价，存在交易机会
- **< 0**：模型认为概率低于市场定价，市场可能高估
- **= 0**：模型和市场一致，没有明显边缘

**边缘强度**（来自 `edge.js` 的决策逻辑）：
- **≥ 0.2（20%）**：STRONG（强烈边缘）
- **≥ 0.1（10%）**：GOOD（良好边缘）
- **< 0.1**：OPTIONAL（可选边缘）

**交易决策阈值**（根据剩余时间调整）：
- **早期（> 10分钟）**：边缘 ≥ 5% 才交易
- **中期（5-10分钟）**：边缘 ≥ 10% 才交易
- **后期（< 5分钟）**：边缘 ≥ 20% 才交易

### 8.3 实际应用

**寻找交易机会**：
1. **正边缘（上涨）**：模型预测 > 市场定价 → 考虑做多
2. **正边缘（下跌）**：模型预测 > 市场定价 → 考虑做空
3. **负边缘**：模型预测 < 市场定价 → 避免交易

**风险控制**：
- 边缘越大，交易价值越高
- 但也要考虑剩余时间和价格差距
- 如果剩余时间 < 2分钟，即使边缘很大，也可能来不及

## 九、市场状态（Regime）

### 9.1 计算原理

**市场状态检测**（来自 `regime.js`）用于判断当前市场是趋势还是震荡。

**状态类型**：
1. **TREND_UP**：上升趋势
   - 条件：价格 > VWAP 且 VWAP斜率 > 0
2. **TREND_DOWN**：下降趋势
   - 条件：价格 < VWAP 且 VWAP斜率 < 0
3. **RANGE**：震荡市场
   - 条件：VWAP频繁交叉（20分钟内≥3次）
4. **CHOP**：横盘整理
   - 条件：低成交量 + 价格接近VWAP

### 9.2 如何解读

**趋势市场**：
- 技术指标更可靠
- 可以跟随趋势
- 避免逆势交易

**震荡市场**：
- 技术指标可能产生假信号
- 适合区间交易
- 需要更谨慎

**横盘市场**：
- 方向不明确
- 等待突破信号
- 避免交易

## 十、综合决策流程

### 10.1 决策步骤（来自 `edge.js`）

1. **计算边缘**：模型预测 vs 市场定价
2. **检查边缘阈值**：根据剩余时间调整
3. **检查模型概率**：确保模型有足够信心
4. **决定行动**：
   - ENTER：入场交易
   - NO_TRADE：不交易

### 10.2 决策条件

**入场条件**（必须同时满足）：
1. 边缘 ≥ 阈值（根据时间调整）
2. 模型概率 ≥ 最小概率（根据时间调整）
3. 有明确的方向（UP或DOWN）

**不交易原因**：
- 边缘太小
- 模型概率不足
- 缺少市场数据
- 方向不明确

### 10.3 实际应用建议

**早期阶段（剩余时间 > 10分钟）**：
- 可以更激进，边缘5%即可
- 有足够时间让价格回归

**中期阶段（5-10分钟）**：
- 需要边缘10%以上
- 时间开始紧迫

**后期阶段（< 5分钟）**：
- 需要边缘20%以上
- 价格变化空间有限
- **非常谨慎**

## 十一、常见陷阱和注意事项

### 11.1 指标滞后

**问题**：技术指标基于历史数据，可能滞后于实际价格
**解决**：结合实时价格变化和市场情绪

### 11.2 过度拟合

**问题**：模型可能过度优化历史数据
**解决**：关注边缘，而不是绝对预测

### 11.3 市场效率

**问题**：如果市场已经充分定价，可能没有边缘
**解决**：等待下一个市场窗口，或寻找其他机会

### 11.4 时间压力

**问题**：15分钟窗口很短，价格可能快速反转
**解决**：
- 早期入场，避免后期追涨杀跌
- 设置止损
- 不要过度交易

### 11.5 流动性风险

**问题**：低流动性可能导致滑点或无法成交
**解决**：检查流动性，避免在低流动性市场交易

---

## 十二、最佳实践

1. **多指标确认**：不要依赖单一指标
2. **寻找边缘**：模型预测 vs 市场定价
3. **时间管理**：根据剩余时间调整策略
4. **风险控制**：设置止损，控制仓位
5. **持续学习**：记录交易结果，优化策略

## 总结

这个系统通过综合多个技术指标，与Polymarket市场定价对比，寻找交易边缘。关键是要理解：

1. **技术指标**：提供预测概率
2. **市场定价**：反映市场共识
3. **边缘**：两者差异 = 交易机会
4. **时间**：影响决策阈值和风险

记住：**没有完美的指标，只有合理的概率和风险管理**。
